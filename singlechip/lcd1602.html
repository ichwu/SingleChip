LCD1602

VSS 电源地
VDD 电源正
VL 液晶显示偏压信号 可接3.3K左右下拉电阻
RS 数据/命令 H/L
R/W 读/写 H/L
E 使能
D0~D7 Data IO
BLA 背光正
BLK 背光负 可接220电阻

液晶内部含RAM 当我们给液晶发送命令/数据时 需要先将其保存在缓存中 然后再写到内部寄存器/RAM中 这需要一定时间 因此在读写操作前需要读一下液晶的状态

液晶内部带了80bytes的显示RAM用来存储数据
第一行0x00到0x0F与液晶第一行16个字符显示位置相对应 第二行同理
液晶能显示ASCII码

液晶内部有一个数据指针 它指向哪里 我们写的那个数据就会送到相应地址
液晶内部有一个状态指针 bit7 bit6~bit0 当bit7为1时表示液晶忙 bit6~bit0为当前数据的指针的值

1602液晶指令


1：写指令 0x38 设置16*2 显示 5*7点阵 8位数据接口
2：0b00001DCB DCB全是1表示 开显示 显示光标 光标闪烁
3：0b000001NS N=1表示读或写一个字符后 指针加1 光标加1
N=0表示读或写一个字符后 指针减1 光标减1
S=1表示写一个字符后（N=1）整屏左移 （N=0）整屏右移
S=0表示写一个字符后 整屏显示不移动
4：0x01 数据指针清零 显示清零
0x02 数据指针清零 显示不清零
5：0b1 + RAM地址 读写数据前都要先设置好地址
先看读操作 因为是读操作 所以R/W要先拉高电平 而不管它原来是读操作还是写操作 RS可以是0/1 当RS和R/W经过tsp1时间后E才升高电平 再过tD时间LCD1602就输出DB有效数据 我们就可以读DB的数据 读完后我们将E拉低 经过一段时间后RS R/W才可以变化 对于时间变化要求 可以在Datesheet中找到
整体上 器件都是有一个最快速度的限制 而没有最慢限制
所以 当换用高速单片机时 需要加入软件延时来满足较慢的时序要求

1602的基本操作时序 一共有4个


首先把用到的总线接口做一个统一声明
#difine LCD1602_DB P0
sbit LCD1602_RS = P1^0;
sbit LCD1602_RW = P1^1;
sbit LCD1602_E = P1^2;

1，读状态（发出命令 我要读状态）
LCD1602_DB = 0xFF; //单片机读外部状态前 必须先保证自己是高电平
LCD1602_RS = 0;
LCD1602_RW = 1;
LCD1602_E = 1;
do {
LCD1602_E = 1; // 打开开关
sta = LCD1602_DB; //将液晶的状态字存到sta
LCD1602_E = 0; //关闭开关
} while (sta & 0x80); //当bit7 = 1 时表示液晶正忙 重复检测直到不为0

2，读数据 （不常用）
RS = H； R/W = H； E = H；

3，写指令
RS = L； R/W = L； D0~D7 = 指令码； E = 高脉冲；
首先要保证E引脚为低电平 1602液晶只要没有接收到E引脚的使能控制 它都不会来读总线上的信号的
1602只要检测到高脉冲中的下降沿 并且检测到RS=L；R/W=L； 就马上来读取D0~D7的数据 完成写指令
4，写数据
RS = H； R/W = L； D0~D7 = 数据； E = 高脉冲；
